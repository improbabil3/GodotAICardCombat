# Piano grafico: indice e ordine di lettura

Questa cartella contiene il piano di lavoro completo per portare la grafica nel progetto.

Non e una sezione `how-to`.

Differenza corretta:
- `docs/how-to/` = guide operative puntuali su file, scene, nodi e script gia esistenti
- `docs/graphics-plan/` = cosa va fatto, in che ordine, con quali vincoli, con quali strumenti e con quali verifiche

Se devi decidere il lavoro da fare, parti da qui.

Se devi poi eseguire un intervento mirato dentro il progetto, vai in `how-to`.

## Scopo dei documenti

### 001 - Piano di lavoro completo

Serve per capire:
- cosa fare prima
- cosa fare dopo
- cosa non fare subito
- dove rischi di sprecare budget

### 002 - Bibbia visiva

Serve per bloccare:
- stile
- palette
- tipografia
- silhouette
- materiali UI
- regole di leggibilita

### 003 - Manifest asset

Serve per sapere:
- quanti asset produrre
- come chiamarli
- dove salvarli
- quali chiavi immagine esistono davvero

### 004 - Pipeline, strumenti e budget

Serve per organizzare:
- software
- installazione
- cartelle sorgenti
- export
- budget massimo 200 dollari
- controlli licenza

### 005 - Integrazione Godot

Serve per sapere:
- quali file toccherai davvero
- in che ordine
- quali nodi oggi sono placeholder
- quali modifiche richiedono refactor prima della grafica finale

### 006 - Prompt AI e verifica

Serve per usare AI senza farti danneggiare la coerenza del progetto.

## Regola pratica finale

Se una domanda inizia con:
- "cosa devo fare prima?"
- "quanti asset servono davvero?"
- "come organizzo il lavoro?"
- "quale software uso?"

la risposta deve stare qui dentro, non in `how-to`.
