# Contributing

Thanks for your interest in contributing to GodotAICardCombat. A few guidelines to help maintain a smooth collaboration:

1. Fork the repository and open a branch named `feat/xxx` or `fix/xxx`.
2. Keep changes small and focused; one feature/bug per PR.
3. Write clear commit messages using the conventional style: `feat(...)`, `fix(...)`, `chore(...)`.
4. When opening a PR, link the related issue (if any) and include screenshots or steps to reproduce.
5. Use the provided issue and PR templates.

Code style

- GDScript files should follow Godot style and use descriptive names.

Testing & QA

- This project currently has no automated tests. If you add tests, include CI steps.
